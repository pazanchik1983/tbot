# Android-клиент (план)

Десктоп-приложение `TBot.exe` уже умеет работать как REST/WebSocket-сервис
(`tbot/api/server.py`). Это намеренное архитектурное решение: **ядро + UI
полностью развязаны через event bus**, поэтому мобильный клиент — это просто
ещё один потребитель той же шины.

## Архитектура

```
   ┌──────────────┐  websocket/REST   ┌─────────────────┐
   │ Android App  │  ←──────────────→ │ TBot core (PC)  │
   │ (Flutter/    │      X-Auth       │  broker + agent │
   │  Kotlin)     │                   └─────────────────┘
   └──────────────┘
```

Бот работает 24/7 на ПК/мини-сервере, телефон только:
- показывает график и список ОФЗ (`GET /instruments`, `/candles`);
- показывает поток сигналов и сделок (`WS /stream`);
- даёт «pause/resume» агента, ручные ордера и смену настроек.

## Рекомендуемый стек

| Что                | Чем                                                   |
| ------------------ | ----------------------------------------------------- |
| UI                 | **Flutter** (один кодбейз iOS/Android)                |
| Графики            | `fl_chart` или `syncfusion_flutter_charts` (свечи)    |
| REST               | `dio`                                                 |
| WebSocket          | `web_socket_channel`                                  |
| Безопасное хранение| `flutter_secure_storage` для общего токена `X-Auth`   |
| Notifications      | `flutter_local_notifications` для сигналов агента     |

## Минимальный набор экранов

1. **Watchlist (ОФЗ)** — список инструментов + последняя цена.
2. **Chart** — свечи + горизонтальные уровни покупок/продаж + маркеры сделок.
3. **Signals/Trades feed** — лента из WS.
4. **Settings** — URL сервера, общий токен (X-Auth), вкл/выкл агента.

## Развёртывание

1. Запустить десктопный бот:
   ```powershell
   python -m tbot.api.server --host 0.0.0.0 --port 8765 --token <SECRET>
   ```
2. Открыть порт 8765 наружу либо ходить через VPN/Tailscale (рекомендуется,
   чтобы не выставлять торговый API в публичную сеть).
3. В мобильном клиенте указать `https://<host>:8765` и `X-Auth: <SECRET>`.

## Почему не «бот прямо в телефоне»?

- Android агрессивно убивает фоновые процессы (Doze, App Standby);
- сеть/батарея/обрывы — высокий риск «зависнуть в позиции»;
- ML-обучение и стрим свечей через grpc — тяжело для мобилки.

Если позже всё-таки потребуется автономный мобильный режим — ядро
переносится через Chaquopy / Kivy / BeeWare, но как **отдельный проект**.
